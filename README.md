# Сервис ИИ-мониторинга: классификация сообщений

Проект для автоматического отнесения входящих текстовых сообщений (постов, комментариев, жалоб) к пользовательским смысловым категориям (например: «мошенничество», «жалоба на ЖКХ», «дороги»).

Главная особенность системы: **категории не зашиты в код**. Пользователь может динамически передавать любой набор категорий и ключевых слов при запуске.

## Архитектура алгоритмов

Согласно концепции, система поддерживает несколько независимых подходов к классификации с единым интерфейсом. 

1. **Алгоритм 1: Точное совпадение (`classify_exact.py`)** — *Реализовано*
   Лемматизация текста сообщения и ключевых слов с помощью NLP-библиотеки `natasha`. Оценка считается по доле нормализованных стоп-слов категории, найденных в тексте.
2. **Алгоритм 2: Расстояние Левенштейна (`classify_levenshtein.py`)** — *В разработке*
   Нечёткое сопоставление (поиск с учетом опечаток и искажений).
3. **Алгоритм 3: Эмбеддинги (`classify_embeddings.py`)** — *В разработке*
   Семантический поиск по косинусной близости векторов (улавливание смысла без точного совпадения слов).
4. **Алгоритм 4: LLM (`classify_llm.py`)** — *В разработке*
   Zero-/few-shot классификация с помощью больших языковых моделей.

## 🛠 Установка и запуск

### 1. Подготовка окружения
Убедитесь, что у вас установлен Python 3.10+.
```bash
# Клонируем репозиторий
git clone https://github.com/b4end/ai-monitor-classification
cd ai-monitor-classification

# Создаем и активируем виртуальное окружение
python -m venv .venv
source .venv/bin/activate  # Для macOS/Linux
# .venv\Scripts\activate   # Для Windows
```

### 2. Установка зависимостей
```bash
pip install -r requirements.txt
```

### 3. Использование
Единый интерфейс позволяет легко импортировать нужный классификатор в необходимые сервисы:

```python
from classify_exact import classify

message = "На дороге огромная яма, нужен ремонт!"
categories = {
    "жкх": ["горячая вода", "отключение", "отопление"],
    "дороги": ["яма", "ремонт дороги", "пробка"]
}

result = classify(message=message, stop_words=categories)

# Результат возвращается в виде Pydantic-модели
print(result.model_dump_json(indent=2))
```

## 🧪 Тестирование

В проекте настроены тесты для проверки краевых случаев (пустые сообщения, пунктуация, капс, отсутствие словарей и пересечение категорий).

Запуск тестов осуществляется из корня проекта. Пример:
```bash
python tests/test_01.py
```
Скрипт автоматически замеряет время выполнения функций и сверяет вывод с эталонным файлом `test_correct.json`.

## 📊 Формат вывода
Все модули возвращают унифицированный ответ:
```json
{
  "message": "На дороге яма",
  "results": [
    {
      "category": "дороги",
      "score": 0.5,
      "matched": ["яма"]
    },
    {
      "category": "жкх",
      "score": 0.0,
      "matched": []
    }
  ]
}
```
Где `score` — нормализованная оценка релевантности от `0.0` до `1.0`.
